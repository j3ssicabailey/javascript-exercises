Introduction to TypeScript

What is TypeScript?

TypeScript is a superset of JavaScript that adds static types. It helps developers catch errors early and write more maintainable code.

JavaScript vs TypeScript

JavaScript TypeScript
Dynamically typed Statically typed
No compilation step required Requires transpilation to JavaScript
Can lead to runtime errors Errors detected at compile time

Example: JavaScript vs TypeScript

JavaScript

// JavaScript (No type checking)
function add(a, b) {
    return a + b;
}
console.log(add(5, "10")); // Outputs: "510" (string concatenation!)

TypeScript

// TypeScript (Static types ensure correctness)
function add(a: number, b: number): number {
    return a + b;
}
console.log(add(5, 10)); // Outputs: 15

Why Use TypeScript?

  • Type Safety: Helps catch errors before runtime.
  • Better Code Readability: Makes it easier to understand codebases.
  • Enhanced IDE Support: Provides better autocomplete and refactoring.

Transpilation

TypeScript code is transpiled into JavaScript using the TypeScript compiler (tsc).

tsc myfile.ts

This generates a JavaScript file that can run in the browser or Node.js.

Linters and Configuration

TypeScript works well with linters like ESLint and has a configuration file (tsconfig.json) to control compilation settings.
