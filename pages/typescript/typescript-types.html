Classes, Interfaces & Types

Classes in TypeScript

Classes define blueprints for objects, supporting encapsulation and inheritance.


class Person {
    name: string;
    age: number;

    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }

    greet(): void {
        console.log(`Hello, my name is ${this.name}`);
    }
}
let user = new Person("Alice", 30);
user.greet();

Interfaces

Interfaces define the structure of an object without implementation.


interface User {
    name: string;
    age: number;
}
let user: User = { name: "Bob", age: 25 };

Type Aliases

Types can be used similarly to interfaces but with more flexibility.


type Employee = {
    name: string;
    role: string;
};
let worker: Employee = { name: "Eve", role: "Engineer" };

When to Use Interfaces vs Type Aliases

  • Use interfaces when defining object shapes that may be extended.
  • Use type aliases for complex types like unions or tuples.
