Understanding Prototypes in JavaScript

In JavaScript, every object has a prototype, which is another object from which it inherits properties and methods. This forms the prototype chain.

Prototype Chain

When accessing a property on an object, JavaScript first looks at the object itself. If the property is not found, it checks the object's prototype, continuing up the chain until it reaches null.

function Person(name) {
    this.name = name;
}
Person.prototype.greet = function() {
    console.log("Hello, my name is " + this.name);
};
const john = new Person("John");
john.greet(); // "Hello, my name is John"

Modifying Prototypes

Prototypes can be extended, but modifying built-in prototypes should be done with caution.

Array.prototype.last = function() {
    return this[this.length - 1];
};
console.log([1, 2, 3].last()); // 3
