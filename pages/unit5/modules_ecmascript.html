Understanding Modules in JavaScript

ES6 introduced modules to enable better code organization and reuse. Modules allow you to split code into separate files and import only what is needed.

Exporting Modules

Use the export keyword to make functions, variables, or classes available in other files.

// math.js
export function add(a, b) {
    return a + b;
}

export const PI = 3.1416;

Importing Modules

Use import to bring in exported values.

// main.js
import { add, PI } from "./math.js";

console.log(add(5, 3)); // 8
console.log(PI); // 3.1416

Default Exports

A module can have a single default export, imported without curly braces.

// greet.js
export default function greet(name) {
    return `Hello, ${name}!`;
}

// main.js
import greet from "./greet.js";

console.log(greet("Alice")); // Hello, Alice!

Renaming Imports and Exports

Aliases can be assigned to avoid naming conflicts.

// utils.js
export { add as sum } from "./math.js";

// main.js
import { sum } from "./utils.js";

console.log(sum(4, 2)); // 6
