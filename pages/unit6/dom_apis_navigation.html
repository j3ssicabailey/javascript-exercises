DOM APIs and Navigation

Node API

The Node API allows interaction with all types of nodes in the DOM, including elements, text, comments, and more. Some key methods include:

  • node.appendChild() - Adds a node as the last child of a parent.
  • node.removeChild() - Removes a child node.
  • node.replaceChild(newNode, oldNode) - Replaces an existing child node.
  • node.cloneNode(deep) - Clones a node. If deep is true, it clones all children as well.

Element API

The Element API extends the Node API and includes methods specific to element nodes. Some commonly used methods:

  • element.setAttribute(name, value) - Sets an attribute on an element.
  • element.getAttribute(name) - Retrieves the value of an attribute.
  • element.removeAttribute(name) - Removes an attribute.
  • element.classList.add(), remove(), toggle() - Modifies class attributes efficiently.

Adjacent Insertion API

This API provides methods to insert HTML at specific positions relative to an element:

  • element.insertAdjacentElement(position, element)
  • element.insertAdjacentHTML(position, htmlString)
  • element.insertAdjacentText(position, text)

The position can be:

  • beforebegin - Before the element itself.
  • afterbegin - Inside the element, before its first child.
  • beforeend - Inside the element, after its last child.
  • afterend - After the element itself.

// Example:
const div = document.querySelector(".container");
div.insertAdjacentHTML("beforeend", "
New paragraph
");

Element vs. Node Navigation

There are two ways to navigate the DOM: by element and by node.

By Element

  • element.children - Returns only element nodes.
  • element.firstElementChild - First child element.
  • element.lastElementChild - Last child element.
  • element.nextElementSibling - Next sibling element.
  • element.previousElementSibling - Previous sibling element.

By Node

  • node.childNodes - Returns all nodes, including text and comments.
  • node.firstChild - First child node (could be text).
  • node.lastChild - Last child node.
  • node.nextSibling - Next node (could be text or comment).
  • node.previousSibling - Previous node.
